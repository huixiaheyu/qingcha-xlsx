# 图片 EXIF 水印工具 — 设计方案

## 概述

纯浏览器端工具，无需服务器。用户选择/拖拽文件夹，读取图片 EXIF 中的 GPS 经纬度，结合文件修改时间和用户输入，在图片左下角添加水印后批量下载。

## 文件结构

```
watermark-tool/
  └── index.html          ← 单文件，浏览器打开即用
```

## 技术栈

| 技术 | 用途 |
|------|------|
| **exifr** (CDN) | 提取 EXIF GPS 经纬度 |
| **JSZip** (CDN) | 批量打包 ZIP 下载 |
| **Canvas API** | 绘制水印，输出图片 |

纯前端，不上传任何数据到服务器。

## 水印内容

```
拍摄时间：2026.06.25 13:16
经度：113.576750°E
纬度：34.810018°N
地块编码：42
村名：刁沟村
```

## 自适应比例规则

所有尺寸参数按图片尺寸动态计算，无硬编码：

| 参数 | 公式 | 举例 (8000×6000) | 举例 (4000×3000) |
|------|------|-------------------|-------------------|
| 字号 | `min(w,h) × 0.02`，下限 12px | **120px** | **60px** |
| 左边距 | `字号 × 1.0` | 120px | 60px |
| 下边距 | `字号 × 1.0` | 120px | 60px |
| 行高 | `字号 × 1.5` | 180px | 90px |
| 背景内边距 | `字号 × 0.4` | 48px | 24px |
| 背景圆角 | `字号 × 0.3` | 36px | 18px |

## 核心逻辑流程

```
用户选择/拖拽文件夹
    ↓
提取文件夹名 → 地块编码
提取文件名   → 保留原名
    ↓ (逐张图片)
读取 EXIF GPS → 无GPS → 标记跳过，继续下一张
读取 file.lastModified → 格式化为 YYYY.MM.DD HH:mm
    ↓ (用户点击"开始处理")
Canvas 渲染：原图 + 左下角水印
    ↓
保持原格式输出（JPEG q=1.0 / PNG 无损）
    ↓
结果按文件夹分组展示 → 下载全部 (ZIP)
```

## 关键实现要点

| 功能 | 实现方式 |
|------|----------|
| 文件夹选择 | `<input webkitdirectory>`（点击）+ `webkitGetAsEntry`（拖拽） |
| GPS 提取 | `exifr` 解析 `GPSLatitude/GPSLongitude` → DMS 转十进制 |
| 拍摄时间 | `file.lastModified` → 格式化为 `YYYY.MM.DD HH:mm` |
| 水印渲染 | Canvas: 先画原图 → 再叠加半透明背景 → 白色文字 |
| 输出格式 | 保持原格式（JPEG q=1.0, PNG 无损） |
| 批量下载 | JSZip 打包，按文件夹结构组织 |

## 边界情况处理

- **无 GPS 数据**：跳过该图片，记录原因，继续处理其余
- **无村名**：水印中村名行显示空值
- **非图片文件**：自动过滤（仅处理 image/\* 类型）
- **超大图片**：Canvas 原生支持大图，按比例字号自动适配
- **浏览器兼容**：`webkitdirectory` / `webkitGetAsEntry` 需 Chromium 内核（Chrome/Edge）
